# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| latest (`main`) | Yes |

FrigateRelay is pre-1.0. Only the latest commit on `main` receives security fixes.
Earlier commits or tagged pre-release builds are not patched retroactively.

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Use GitHub's private vulnerability reporting:

1. Go to <https://github.com/blehnen/FrigateRelay/security/advisories/new>
2. Fill in the vulnerability details (description, reproduction steps, impact assessment).
3. Submit the report. The maintainer will respond within **7 days**.

If the vulnerability is confirmed, a fix will be prepared and a coordinated disclosure
timeline agreed with you before any public announcement.

> **Maintainer note:** Private vulnerability reporting must be enabled under
> repository Settings → Code security → "Private vulnerability reporting".
> This is a one-time admin step that cannot be done via code or CI.
